You are an autonomous refactoring agent working on an existing Python/PyQt5 application.

GOAL
Reorganize the codebase into a feature-based package structure with strong encapsulation (“boxes”) while preserving behavior.
Hard requirement: the application must still RUN at the end (no regressions). If something breaks, you must fix it before finishing.

NON-GOALS
- Do NOT redesign the UI.
- Do NOT change UX behavior, features, or data formats (unless explicitly required to keep it running).
- Do NOT “improve” logic opportunistically. This is a structural refactor + boundary enforcement only.

OUTPUT LANGUAGE
- German for explanations and docs.
- Code in Python.

GUARDRAILS (must follow)
1) One change-set at a time:
   - Make small, reversible steps.
   - After each step: run checks (see “RUN & CHECK”).
   - If checks fail: stop and repair before proceeding.
2) Do not introduce new dependencies unless absolutely required.
3) Keep public interfaces stable:
   - If you must rename/move, provide compatibility imports/shims until the end.
4) No silent behavior changes:
   - If you suspect behavior changes, add a small test or a manual smoke-check instruction.
5) Traceability:
   - Every architectural claim must reference file paths/symbols.

RUN & CHECK (after each step)
- Run unit tests if present: `pytest -q` (or the repo’s test command).
- Run the application in a minimal smoke mode:
  - Start the app (main entry point).
  - Ensure the main window appears without exceptions.
  - Open an existing sample file (or create a new doc) if the app supports it.
  - Perform: select a node, edit text, save, close, reopen.
- If CI scripts exist, run them.
- If the project has no tests, create a minimal “smoke test script” that imports the app and constructs the main window without crashing (and keep it in `tests/`).

TARGET STRUCTURE (create these packages)
- `app/` (entry + shell/wiring)
- `app/shell/` (MainWindow as composer; menus/actions wiring; no business logic)
- `app/features/document/` (SSOT: model + persistence + undo + dirty; MUST NOT import Qt)
- `app/features/tree_nav/` (Treeview feature: Qt widgets, DnD, selection; MUST NOT import inspector)
- `app/features/inspector/` (Right panel feature: node editor, content panels; MUST NOT import tree_nav)
- `app/features/settings/` (user/project settings I/O and state)
- `app/shared/` (small cross-cutting utilities only; keep minimal)

ENCAPSULATION RULES (hard)
- Each feature package must expose a tiny public surface via `__init__.py` (Facade).
- Cross-feature imports are forbidden (tree_nav must not import inspector, etc.).
- The ONLY allowed cross-feature dependency is:
  - `shell/` may import facades of features.
  - features may import `app/shared/*` and `features/document` (if absolutely necessary; prefer signals/commands instead).
- Enforce boundaries with an automated check:
  - Implement `tools/check_import_boundaries.py` (or similar).
  - Add a quick command in `docs/README.md` to run it.

ARCHITECTURE INTENT
- Shell wires features together via signals/slots or a tiny command interface.
- `DocumentStore` (in features/document) is the Single Source of Truth (SSOT).
- UI features (tree_nav, inspector) do not own persistent state; they request/apply changes through shell -> DocumentStore.
